# Product Context

## Why it exists
- Streamline access to university counseling services and announcements.
- Enable students to schedule and track appointments.
- Provide admin/counselor tools to manage counseling operations.

## Target users
- Students (end users)
- Counselors
- Administrators

## Core user journeys
- Student: open app → view landing/services → log in → dashboard → schedule/view appointments → view announcements → manage profile.
- Counselor: log in → dashboard → manage appointments/announcements (future screens).
- Admin: log in → dashboard → manage admins/announcements/appointments (future screens).

## UX goals
- Fast navigation with named routes.
- Minimal friction on common flows (appointments, announcements).
- Consistent theming and typography across platforms.

## Constraints
- Backend base URL may vary by environment; device/emulator/web differences.
- Network reliability and latency.
